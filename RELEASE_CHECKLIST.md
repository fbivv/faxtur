# Checklist de release Faxtur

Avant publication GitHub :

- [ ] Installer Faxtur sur un poste vierge.
- [ ] Vérifier que l'installation ne demande pas les droits administrateur.
- [ ] Vérifier que les dossiers par défaut sont bien sur le Bureau de l'utilisateur.
- [ ] Convertir une facture de référence.
- [ ] Valider localement avec veraPDF.
- [ ] Valider une facture sur un validateur Factur-X externe.
- [ ] Vérifier que le logo apparaît dans la fenêtre et le raccourci.
- [ ] Vérifier que les fichiers utilisateur ne sont pas écrasés lors d'une mise à jour.
- [ ] Créer le tag Git : `v1.0.2-rc1`.
- [ ] Créer une release GitHub et joindre l'archive ou l'installateur.
